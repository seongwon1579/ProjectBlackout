# PPT 작성 가이드

## 작성 요령
### 제출 형태
- 팀별 프로젝트 결과보고서는 PPT 형태로 작성하여 제출해야 함

### 파일명
- 파일명은 '결과보고서_2팀(언리얼 엔진을 활용한 멀티플레이 게임 구현) 형태로 생성되어야 함

### 구성 내용
- 이하의 목차 항목/구성/세부내용이 *모두 포함되어야* 함
- 목차 항목은 다음 5개를 중심으로 작성하되 필요에 따라 더 추가할 수 있음
1. 프로젝트 개요
    1. 프로젝트 주제 및 선정 배경, 기획 의도가 포함되어야 함
    2. 프로젝트 구현 내용, 컨셉, 훈련 내용과의 연관성 등이 포함되어야 함
    3. 개발 환경에 대한 설명이 포함되어야 함
    4. 프로젝트의 목적과 프로젝트 구조, 진행 일정, 구체적인 실행(구현) 내역 등이 포함되어야 함
    5. 프로젝트 산출물의 기대 효과 (언리얼 엔진 활용 역량 강화 쪽으로 어필) 등이 포함되어야 함

2. 프로젝트 팀 구성 및 역할
- 프로젝트에 참가한 *훈련생 별로 주도적으로 참여한 부분을 중심*으로 작성한다.
- 프로젝트에 참가한 훈련생은 다음과 같다:
| 이름 | 역할 (팀장, 팀원, 멘토) | 담당 업무 |
김민영 | 팀장 | Core, GAS, UI
최승현 | 팀원 | Server, Game Flow
조성원 | 팀원 | AI, Animation
허혁 | 팀원 | Player, Animation
* 각 훈련생이 참여한 부분은 https://github.com/ThunderVolt45/ProjectBlackout 내의 PR 기여 내역을 참고하여 첨삭되어야 한다.

3. 프로젝트 수행 절차 및 방법
- 프로젝트의 *사전 기획*과 *프로젝트 수행 및 완료 과정*으로 나누어 작성한다.
- 프로젝트 수행 절차를 도식화하여 제시한다.
- 실제 프로젝트를 수행한 세부적인 기간과 활동 내용을 작성한다.

4. 프로젝트 수행 경과
- 프로젝트의 *주요 기능들을 상세히 작성*한다.
- 프로젝트의 주요 기능들을 구현하는데에 *활용된 기술 및 구현 방법*을 상세히 작성한다.
- 프로젝트 수행 결과물을 잘 드러낼 수 있도록 *클래스 다이어그램 등의 이미지*를 첨부하여 작성한다.

5. 자체 평가 의견
- 프로젝트 결과물에 대한 프로젝트 기획 의도와의 부합 정도 및 실무 활용 가능 정도, 달성도, 완성도 등 *훈련생의 자체적인 평가 의견과 느낀 점을 작성*한다.
- PPT를 작성하는데 사용할 의견은 다음과 같다:
* 사전 기획의 관점에서 *프로젝트 결과물에 대한 완성도 평가* (7/10)
* 개인 또는 우리 팀이 *잘한 부분과 아쉬운 점*
  - 잘한 점: AI 에이전트들을 개발 프로세스에 성공적으로 통합해 빠르고 정확한 결과물 산출이 가능했음. 주요 기능들을 제한된 에셋과 시간 안에 양호한 수준으로 구현해내었음.
  - 아쉬운 점: 부족한 에셋으로 인해 일부 기획이 잘려나가야 했음. 프로젝트 일정 관리 실패로 인해 결과물의 퀄리티 보장에 한계가 있었음.
* 프로젝트 결과물의 *추후 개선점이나 보완할 점*
  - 프로젝트 일정 관리 방법론의 학습과 적용이 필요함
  - 프로젝트의 문서화 방법론의 학습과 적용이 필요함
  - 주요 코드의 미사용 코드 정리, 가독성 개선, 유지보수성 개선 등 전반적인 리펙토링이 필요함
* 프로젝트를 수행하면서 *느낀 점이나 경험한 성과*
  - 기획 -> 구현 -> 테스트 -> 폴리싱 등 게임 개발의 전 과정을 경험함으로써 클라이언트 개발자 직종에 대한 전반적으로 이해할 수 있었음
  - 협약 기업의 멘토링을 통해 현재 업계에 대한 현업자들의 시각과 현재 업계가 추구하는 방향성을 엿볼 수 있었음
